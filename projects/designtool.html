Project Overview

The Unity Design Tool is a custom toolkit I developed from scratch to address a critical gap in XR development workflows. This tool helps designers and developers quickly prototype and visualize spatial interactions in Unity, with a specific focus on reducing cognitive load during the design process.

The Challenge

Extended Reality (XR) design introduces unique spatial and interaction challenges that traditional design tools struggle to address. When creating immersive experiences, designers often face these key problems:

  • Difficult visualization of 3D interactions and spatial relationships
  • Steep learning curve for technical tools that interrupts creative flow
  • Lack of specialized tools to evaluate cognitive load and interaction complexity
  • Inefficient iteration cycles between design and development

The most challenging aspect of XR design isn't creating individual elements—it's understanding how these elements work together in three-dimensional space and how users will perceive and interact with them. This tool bridges the gap between creative vision and technical implementation.

Key Features

Spatial Prototyping
Quickly sketch and manipulate 3D interfaces and interaction zones with intuitive controls inspired by Gravity Sketch's spatial modeling approach.
Perception Visualization
Analyze user fields of view and attention patterns with heat maps and gaze prediction tools to optimize element placement.
Cognitive Load Analyzer
Evaluate the mental demands of interactions using metrics based on Fitts' Law and other HCI principles to identify potential usability issues.
Component Generation
Automatically generate Unity components and basic scripts from design prototypes to accelerate the development workflow.

Technical Implementation

The tool was built as a custom editor extension for Unity, leveraging several technologies to create an intuitive design environment:

Unity C#
Custom Editor GUI
ScriptableObjects
MVVM Architecture

I developed the tool using Unity's Editor Extension API, creating custom editor windows, inspectors, and scene view tools. The architecture follows an MVVM (Model-View-ViewModel) pattern to maintain a clean separation between the design data and its visualization.

For the cognitive load analysis features, I implemented algorithms based on established HCI research, incorporating factors such as target size, distance, and occlusion to estimate interaction complexity.

Design Process

The development of this tool was driven by my own frustrations and observations while working on XR projects. The process followed these key stages:

  1. Research: Analyzed existing XR design workflows and identified pain points through interviews with designers and developers
  2. Conceptualization: Created a feature map focused on solving the most critical challenges in spatial interface design
  3. Prototyping: Developed core functionality using rapid prototyping techniques within Unity
  4. User Testing: Gathered feedback from fellow XR designers and developers through structured testing sessions
  5. Iteration: Refined the tool based on user feedback, focusing on usability and workflow integration
  6. Documentation: Created comprehensive guides and examples to facilitate adoption

Applications & Impact

The Unity Design Tool has been used across several XR projects, significantly improving design efficiency and output quality:

  • Reduced prototyping time by approximately 40% compared to standard Unity workflows
  • Enabled non-technical designers to create spatial mockups without requiring developer assistance
  • Improved communication between design and development teams by providing a common visual language
  • Helped identify and resolve potential usability issues early in the design process

The tool has proven particularly valuable for projects with complex spatial interactions, such as VR training simulations and AR interfaces where attention management and cognitive load are critical factors in user experience.
