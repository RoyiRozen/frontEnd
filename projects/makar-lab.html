MakAR Lab - AR Guidance for Maker Spaces

Course: INFO 5358: 3D Interaction Design

Project Overview

MakAR Lab is an augmented reality (AR) system designed for the Cornell Tech MakerLab community, utilizing Meta Orion Smart Glasses. It aims to provide students, especially those new to advanced equipment, with intuitive, real-time, step-by-step guidance. The system also features an admin dashboard, empowering lab managers and "SuperMakers" to easily update instructional protocols, thereby enhancing safety, fostering independent learning, and reducing the support burden on staff.

The Challenge

New MakerLab users often struggle to recall detailed operational procedures for complex machinery after initial orientation. This can lead to hesitation, errors, or an over-reliance on staff assistance. The challenge was to create an accessible, hands-free AR solution that offers contextual guidance directly within the user's field of view, alongside a simple content management system for administrators.

Our Solution: Key Features

We developed a comprehensive AR experience encompassing three core ideas:

Student AR Guide:

  • Step-by-Step Tutorials: Real-time, AR-overlaid instructions for specific equipment (e.g., laser cutter).
  • Interactive Elements: Hand-tracking and gaze-based interactions to navigate tutorials and access information.
  • Contextual Menus: On-demand access to detailed instructions, video tutorials, and a community feed for inspiration and sharing work.
  • Intuitive Interface: Designed to minimize visual obstruction and cognitive load, adhering to Hick-Hyman's Law.

Collaborative AR Design Space:

  • A shared AR environment allowing multiple users (in-person or remote) to collaboratively view and interact with 3D models and share design files.

Admin Dashboard:

  • Easy Content Management: An intuitive interface for non-technical admins to upload and update instructional materials (text, videos, PDFs).
  • AI Machine Detection: Automatically identifies lab equipment to simplify the process of anchoring AR instructions in the 3D space.
  • Instruction Grouping: Allows admins to link instructions for multi-step tasks involving different machines.

Design & Prototyping Process

Requirements Extraction (Part 1):

  • Conducted contextual inquiries, including interviews with the MakerLab Director and students, to understand user needs, pain points, and existing workflows.
  • Developed user personas and problem scenarios.
  • Defined functional, performance, and interface requirements for the system.

Ideation & Design (Part 2):

  • Brainstormed design alternatives based on abstracted requirements for functionality, performance, and interface.
  • Created annotated storyboards to visualize key user scenarios:
    • User entering the lab and selecting equipment via an AR menu.
    • Step-by-step AR tutorial for using the laser cutter.
    • Multi-user collaborative AR design.
    • Admin updating tutorials via the dashboard.
  • (Note: Visual material for storyboards was permitted to use Generative AI, clearly noted.)

Interaction Techniques:

  • Focused on hands-free interaction using gaze-tracking, grab-and-pinch, and pointing gestures (Meta Orion compatible).
  • Incorporated bimanual interactions for collaborative tasks.
  • Emphasized multimodal feedback (visual cues, voice narration) and context-sensitive, translucent AR overlays to maintain situational awareness and minimize cognitive load.

Prototyping (ShapesXR)

Developed a mid-fidelity, interactive prototype for both the user-facing AR application and the admin dashboard.

User App Prototype:

Demonstrated procedure selection, step-by-step guidance with hand-tracking triggers, interactive information tabs (instructions, tutorials, community feed), and a collaboration mode.

Admin Dashboard Prototype:

Showcased AI machine detection visualization, instruction input panels (keyboard/voice), and instruction grouping features.

Evaluation & Iteration

Method:

Conducted a Cognitive Walkthrough due to time constraints and the prototype's fidelity.

Key Findings (User App):

  • Strengths: Liked the equipment selection menu, clear numbered steps hovering over focus areas, and the community project feed.
  • Improvements: Clarify roles in multiplayer view; enhance remote user interaction with real-world models.

Key Findings (Admin Dashboard):

  • Strengths: Appreciated the ability for admins to easily upload new tutorials.
  • Improvements: Suggested a hybrid PC/VR input for extensive typing/content sourcing, with VR for placement.

Learning & Outcomes

This project provided valuable experience in the end-to-end design process for 3D User Interfaces, from contextual inquiry and requirements gathering to iterative prototyping and evaluation. We successfully demonstrated a feasible AR solution that addresses real-world challenges in a MakerLab environment, focusing on user-centric design and intuitive interaction.
