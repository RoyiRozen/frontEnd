Old NYC HCI Course Project

2024
UX/UI Mobile Web

Project Overview

Old NYC was developed as a final project for a Human-Computer Interaction (HCI) course, focusing on designing a cross-platform community-based application with a consistent, clear, and engaging user experience. The platform enables users to share and experience each other's stories and highlights from across New York City, creating a vibrant digital community tied to physical locations.

Digital platforms can strengthen our connection to physical places by layering them with personal narratives. By designing interfaces that are simple, intuitive, and consistent across devices, we create experiences that feel natural and integrated into users' lives.

Design Challenge

The primary challenge of this project was to create a cohesive user experience across multiple platforms while addressing several key requirements:

  • Developing a location-based platform that feels intuitive on both mobile and desktop interfaces
  • Designing a system that encourages community participation and storytelling
  • Creating a consistent visual language and interaction patterns across all touchpoints
  • Balancing simplicity with the rich functionality needed for location-based content sharing
  • Implementing appropriate privacy controls for a platform centered around personal stories

The course framework required us to apply formal HCI principles and methodologies throughout the design process, from initial research through to final evaluation.

Cross-Platform Design

A core focus of the project was maintaining design consistency while optimizing for different devices and contexts:

Mobile Experience
Optimized for on-the-go discovery with location-based notifications and single-handed navigation
Tablet Experience
Hybrid interface balancing rich content exploration with portable form factor
Desktop Experience
Expanded view for deeper exploration and content creation with advanced tools

Each platform shares a common design language while adapting to the unique capabilities and constraints of the device. This approach ensures users can seamlessly switch between devices while maintaining familiarity with the interface.

Key Features

Old NYC was designed with several core features to support its community storytelling mission:

  • Interactive Map: Geospatial interface showing stories and highlights across the city
  • Story Creation: Simple tools for adding text, images, audio, and video tied to specific locations
  • Thematic Collections: Curated groups of stories around neighborhoods, themes, or time periods
  • Discovery Features: Personalized recommendations based on interests and location
  • Community Interaction: Tools for commenting, sharing, and building upon others' stories
  • Guided Tours: User-created or algorithmically-generated paths through connected stories

Design Principles

Throughout the development process, we adhered to several key HCI principles to ensure a high-quality user experience:

Consistency
Common interaction patterns, visual elements, and terminology across all platforms to reduce cognitive load and create a seamless experience.
Progressive Disclosure
Presenting information and controls in a layered approach, revealing complexity only when needed to maintain simplicity without sacrificing functionality.
Contextual Relevance
Adapting the interface and content based on the user's location, history, and current task to provide the most relevant experience.
Feedback & Visibility
Clear system status indicators and immediate feedback for all user actions to create a sense of control and understanding.

These principles were regularly evaluated through user testing sessions and expert reviews throughout the design process.

Technical Approach

The project implementation utilized several key technologies to support its cross-platform design goals:

Responsive Design
Mapping APIs
Design System
Geolocation

Rather than building separate applications for each platform, we implemented a responsive web application with progressive enhancement for mobile devices. This approach allowed us to maintain a single codebase while optimizing the experience for each form factor.

A comprehensive design system was developed with reusable components that could adapt to different screen sizes and interaction models while maintaining visual consistency. This included a flexible grid system, adaptive typography, and context-aware UI components.

User Research & Testing

Following HCI methodologies, our design process included several phases of user research and testing:

  1. Initial Interviews: Conducted with NYC residents and visitors to understand how they currently discover and share city experiences
  2. Competitive Analysis: Review of existing location-based platforms and community storytelling applications
  3. Prototyping: Development of low and high-fidelity prototypes for key user flows
  4. Usability Testing: Formal evaluation sessions with potential users across multiple devices
  5. Iterative Refinement: Several cycles of design improvements based on testing feedback

The testing revealed several key insights that influenced our final design, particularly around the importance of offline access for location-based content and the need for robust privacy controls when sharing personal stories.

Course Outcomes

Old NYC successfully demonstrated the application of HCI principles to the design of a cross-platform community application. The project received positive feedback for:

  • Strong consistency across different devices while maintaining platform-appropriate interactions
  • Clean, engaging interface that prioritized content discovery and community interaction
  • Thoughtful implementation of location-based features that enhanced the storytelling experience
  • Well-documented design process that showcased the application of formal HCI methodologies

Through this project, we gained valuable experience in cross-platform design, location-based services, and community-focused application development—all skills that transfer well to professional UX/UI work.

Learning & Reflection

The Old NYC project provided several important lessons about effective interface design:

  • The importance of maintaining core interaction patterns across platforms while respecting platform-specific conventions
  • Challenges in designing for variable contexts, especially for location-based applications used in diverse environments
  • The value of iterative testing on actual devices rather than just screen simulations
  • The delicate balance between community openness and necessary privacy/moderation controls

These insights continue to influence my approach to design projects, particularly when working on applications that need to function across multiple devices and contexts.
